# Habit Tracker — Android Setup Guide

## Prerequisites
- Android Studio Hedgehog (2023.1.1) or newer
- JDK 17
- Firebase project

## Firebase Setup (Required Before Building)

1. Go to [Firebase Console](https://console.firebase.google.com)
2. Create a new project called "HabitTracker"
3. Add an Android app:
   - Package name: `com.habittracker.app`
   - App nickname: Habit Tracker
4. Download `google-services.json` → place it in `app/google-services.json`
5. Enable **Google Sign-In**: Authentication → Sign-in methods → Google → Enable
6. Create **Firestore Database**: Firestore Database → Create database (start in test mode)
7. Apply security rules (Firestore → Rules tab):
```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId}/{document=**} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
  }
}
```
8. Enable **Cloud Messaging**: Project Settings → Cloud Messaging
9. Add SHA fingerprints: Project Settings → Your app → Add fingerprint
   - Run: `keytool -list -v -keystore ~/.android/debug.keystore -alias androiddebugkey -storepass android -keypass android`

## Open in Android Studio

1. Open Android Studio
2. File → Open → select `C:\Users\Admin\habit-tracker-android`
3. Wait for Gradle sync to complete
4. Connect Android device (API 26+) or start emulator
5. Run `app` configuration

## Notes

- The `default_web_client_id` string is auto-generated by the Google Services Gradle plugin from `google-services.json` — you do not need to add it manually.
- Notifications use `AlarmManager.setExactAndAllowWhileIdle`. On Android 12+ the app requests `SCHEDULE_EXACT_ALARM` permission.
- First run will ask for `POST_NOTIFICATIONS` permission (Android 13+) during the Setup Wizard.
